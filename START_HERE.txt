HOW TO USE (3 SIMPLE COMMANDS)
════════════════════════════════════════════════════════════════════════════

Step 1: Install Dependencies (2 minutes)
────────────────────────────────────────
cd ai_flashcard_generator
pip install -r requirements.txt

Step 2: Train Model (30-60 minutes on GPU, 2-3 hours on CPU)
──────────────────────────────────────────────────────────
python train.py

What happens:
✓ Downloads SQuAD v1.1 dataset (87,599 training examples)
✓ Fine-tunes T5-small model for 3 epochs
✓ Saves trained model to flashcard_t5/ directory
✓ Creates metadata.json with training info

Step 3: Launch Web Interface
──────────────────────────
streamlit run app.py

Then open: http://localhost:8501

USING THE APPLICATION
════════════════════════════════════════════════════════════════════════════

1. Enter your text (study notes, articles, etc.)
2. Adjust settings (optional):
   • Number of flashcards: 6-15
   • Beam search quality: 3-8
3. Click "Generate Flashcards"
4. Flip cards to reveal answers
5. Download as JSON

TECHNICAL SPECIFICATIONS
════════════════════════════════════════════════════════════════════════════

Model Architecture
───────────────────
• T5-small (60 million parameters)
• Transformer encoder-decoder
• 32,128 token vocabulary
• Source: Google T5 base model

Training Configuration
──────────────────────
• Dataset: SQuAD v1.1
• Training examples: 87,599
• Validation examples: 10,570
• Epochs: 3
• Batch size: 16
• Learning rate: 5e-5
• Optimizer: AdamW
• Warm-up steps: 500

Generation Parameters
─────────────────────
• Beam search: 6 beams
• Max length: 256 tokens
• No repeat n-gram: 3
• Temperature: 0.7
• Top-p: 0.9
• Early stopping: True

System Requirements
───────────────────
Minimum:
• Python 3.8+
• 8GB RAM
• 5GB disk space
• CPU or GPU

Recommended:
• Python 3.9-3.11
• 16GB RAM
• 10GB disk space
• NVIDIA GPU with CUDA 11.8+

TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════════

"ModuleNotFoundError" when running scripts
→ Fix: pip install -r requirements.txt

"Model not found" error in app
→ Fix: python train.py (run once first)

CUDA out of memory error during training
→ Fix: Edit train.py, change per_device_train_batch_size=8

Very slow flashcard generation
→ Fix: Reduce num_beams from 6 to 4 in web UI
→ Fix: Use GPU if available

"Not enough context" warning
→ Fix: Use longer input text (>500 characters)

Low quality flashcards
→ Fix: Use well-structured, detailed input text
→ Fix: Increase num_beams to 8
